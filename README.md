# Zotero DataCheck

Zotero DataCheck 是一个面向 Zotero 的插件项目，目标是对论文表格中的数值做基础的统计规律审查，帮助用户快速发现值得进一步人工复核的可疑痕迹。

## 目标能力

- 对选中的文献执行表格数值审查
- 检测常见的可疑模式，包括重复、近重复、平移或缩放痕迹、异常平滑或异常单调趋势
- 对适用的数据执行本福特定律检验
- 对提取出的 p 值执行分布审查，例如显著性阈值附近的堆积现象
- 输出可解释的报告，说明检测依据、命中的单元格或序列，以及该检测的适用条件和局限

## 第一阶段边界

第一阶段优先实现“人工确定目标表格 + 可解释检测 + 可回放报告”。

- 不做 OCR
- 用户在 Zotero PDF 阅读器中手动框选需要分析的表格区域
- 插件从 PDF 的文本层和版面几何信息中提取框选区域内容
- 对提取结果进行表格重建、数值标准化和统计审查

## 第二阶段方向

第二阶段再实现从 PDF 中自动定位并提取表格。

- 优先利用 arXiv、会议、期刊论文中较规范的 PDF 版面和文本层
- 优先做基于文本块、对齐关系和线框信息的自动表格识别
- 不把 OCR 作为默认前提，只有在确实遇到扫描版或图片型 PDF 时才另行评估

## 技术路线

- 基于 zotero-plugin-template 搭建插件骨架
- 插件的正式发布形态是单个 xpi 文件，面向普通 Zotero 用户直接安装使用
- Zotero 集成、PDF 阅读器交互、表格重建、数值标准化、统计审查和结果展示全部由 TypeScript 实现
- 核心统计逻辑尽量写成与 Zotero API 解耦的纯函数模块，方便测试和后续阈值校准

之所以采用这一结构，是因为插件的目标是“单 xpi 即装即用”。这要求正式功能不能依赖外部 Python 服务或额外运行时。Python 仍然可以在开发阶段用于原型验证，但不进入最终交付架构。

## 计划阶段

1. 将模板内容复制到本仓库并完成插件元信息重命名
2. 实现 Zotero PDF 阅读器中的手动框选与区域提取
3. 建立前端内部的数据模型、表格重建模块和结果报告格式
4. 实现第一批检测器：重复、平移或缩放模式、异常趋势、本福特定律、p 值分布
5. 接入 Zotero 菜单、批处理入口、报告面板与结果导出
6. 在下一阶段实现自动 PDF 表格定位与提取
